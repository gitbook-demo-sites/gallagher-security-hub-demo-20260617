# Localization with Crowdin

Gallagher can keep one English source of truth in GitBook and use a localization workflow for supported languages.

## Suggested localization model

| Content type | Recommended flow |
| --- | --- |
| Product concepts | Translate after technical review |
| Release notes | Translate after release manager approval |
| Security advisories | Translate with priority workflow and legal review |
| Partner commercial material | Translate only for active partner regions |
| SDK and API references | Keep code identifiers stable, translate explanatory copy |

## Demo talking points

- Git Sync gives translation teams a stable markdown source.
- Adaptive conditions can stay in the source content and travel with localized pages.
- Crowdin can be positioned as part of the workflow around review, translation memory, and approvals.
